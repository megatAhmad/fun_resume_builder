# Getting Started with the Resume Lifecycle System

This guide will help you set up and run the local-first Resume Lifecycle Repository. The application consists of a Python FastAPI backend and a React (Vite) frontend.

## Prerequisites

- **Python 3.11+**
- **Node.js 18+** & **npm**
- An **OpenRouter API Key** (for the JD Alignment / Gap Bridge features)

---

## 1. Environment Setup

### OpenRouter API Key
The JD alignment agent uses OpenRouter to communicate with language models (defaulting to Anthropic's Claude 3 Haiku).

You must set this as an environment variable before starting the backend:

**Linux/macOS:**
```bash
export OPENROUTER_API_KEY="sk-or-v1-..."
```

**Windows (PowerShell):**
```powershell
$env:OPENROUTER_API_KEY="sk-or-v1-..."
```

*(Optional) You can also customize the model or the database path:*
```bash
export RESUME_MODEL="anthropic/claude-3-haiku"
export RESUME_DB_PATH="resume.db"
```

---

## 2. Backend Setup (FastAPI)

Open a terminal in the root directory of the repository.

1. **Create and activate a virtual environment:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use: venv\Scripts\activate
   ```

2. **Install Python dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
   *Note: The first time the backend runs an embedding, it will download the ~90MB `all-MiniLM-L6-v2` model locally.*

3. **Start the backend server:**
   ```bash
   uvicorn main:app --reload
   ```
   The backend will now be running at `http://127.0.0.1:8000`.

---

## 3. Frontend Setup (React/Vite)

Open a **second** terminal window and navigate to the `frontend` directory.

1. **Install Node dependencies:**
   ```bash
   cd frontend
   npm install
   ```

2. **Start the frontend development server:**
   ```bash
   npm run dev
   ```
   The frontend will now be running, typically at `http://localhost:5173`.

---

## 4. Using the Application

Open your browser and navigate to the frontend URL (`http://localhost:5173`).

### Core Workflows

1. **Add Experience / Add Project:**
   Navigate to these tabs to interact with the **Clarifier Agent**. It will ask you step-by-step questions to ingest your work history, automatically prompting you to add metrics to bullets that lack them.

2. **List Data:**
   The `List` tab displays your actively stored experiences and projects, along with tags indicating which bullets contain metrics or were generated by an agent.

3. **Align Job Description:**
   Go to the `Align JD` tab and paste the text of a job description you are interested in. The **Gap Bridge Agent** will:
   - Extract the core requirements.
   - Perform a semantic search against your local `resume.db` to find matches.
   - For partial matches, it will proactively ask you questions to help bridge the gap and draft a new resume bullet for you.

4. **Settings:**
   Configure the LLM's API retry behavior and wait times directly from the UI. These settings are saved locally to `config.json`.

---

## 5. Running Tests

To verify that the system is working correctly, you can run the backend test suite:

```bash
# Ensure your virtual environment is active

```
